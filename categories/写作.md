## 23. 思辨式写作：观点对立强化（Steelman Argument）

**Prompt：**
```
I believe [主张]。我可能有偏见。

请构建最强有力的反对论点（Steelman Argument），规则：
- 不得使用稻草人论证
- 使用最强的可用证据，即使令人不适
- 不要用"当然你的观点也有道理"来和稀泥
- 结尾必须指出："你当前思维中最薄弱的一点是 X"
```
> 来源：NovaKit — Steelman Argument Prompt (2026)
> 用法：避免AI两面讨好，强制构建最强反对论点，帮你发现思维盲区。

## 26. 跨文化文案本地化策略

**Prompt：**
```
Adapt this copy for a [目标市场/文化] audience.

Original copy: [粘贴原文]
Language: [源语言 → 目标语言]
Target audience cultural context: [文化特征描述]
Product/Service: [产品或服务]
Brand voice: [品牌声音描述]

Do NOT just translate. Instead:

1. Cultural sensitivity scan — flag any idioms, references, humor, or assumptions that won't translate
2. Value proposition reframe — how does the core benefit land differently in this culture?
3. Tone adaptation — adjust formality, directness, emotional appeal
4. Visual/imagery suggestions — culturally appropriate metaphors and examples
5. Localized version — rewrite the copy as if it were originally written in the target language
6. Back-translation — a literal English version so I can verify meaning wasn't lost

Key cultural dimensions to consider: individualism vs collectivism, high vs low context, power distance, uncertainty avoidance.
```
> 📅 2026-07-12
> 来源：Localization / Prompts Lab (2026)
> 用法：超越逐字翻译，做文化适配——调整语气、隐喻和价值观表达以适应目标文化。

## 27. 技术文档体验审查

**Prompt：**
```
Review this technical documentation for user experience and clarity.

Document: [粘贴文档内容]
Target audience skill level: [初级/中级/高级]
Document type: [API文档/用户手册/教程/README]

Evaluate on these dimensions:
1. Information architecture — is the structure logical? Are related concepts grouped?
2. Discoverability — can a user find what they need in < 30 seconds?
3. Clarity — are there unclear terms, ambiguous instructions, or missing definitions?
4. Completeness — are all edge cases, error states, and prerequisites covered?
5. Consistency — are terminology, formatting, and tone consistent throughout?
6. Accessibility — is it screen-reader friendly? Are code examples accessible?

For each issue, provide: (a) exact location, (b) the problem, (c) suggested fix.
Finish with a "quick wins" list of 3 changes that would have the highest impact.
```
> 📅 2026-07-12
> 来源：TechWriting / Prompts Lab (2026)
> 用法：六维技术文档审查，不只看语法错误，还关注信息架构和用户体验。
