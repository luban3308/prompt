## 4. 提示词版本管理策略

**Prompt：**
```
Help me set up a prompt version control system.

Current prompt management: [如何管理提示词]
Number of active prompts: [多少提示词在用]
Team size: [多少人使用/协作]
Prompt changes frequency: [修改频率]
Use cases: [提示词应用场景]

Design a version management strategy:

1. Prompt file structure — how to organize prompts by purpose, version, date
2. Version numbering — semantic versioning for prompts (MAJOR.MINOR.PATCH)
3. Changelog format — what to log per change (what changed, why, effect on output)
4. Prompt testing — how to test a new version against a set of test inputs
5. Rollback procedure — how to revert to a previous known-good version
6. A/B testing prompts — compare two versions side by side on the same input
7. Collaboration — how to review prompt changes in a team
8. Prompt diff — what changed between versions (not just "updated prompt")

Provide a prompt file template with version header and changelog.
```

## 5. 链式思维退火（Chain-of-Thought Annealing）

**Prompt：**
```
Use progressive reasoning depth for this problem.

Problem: [复杂问题/推理任务]

Apply Chain-of-Thought Annealing:

Phase 1 — Quick pass: Give your best direct answer without showing reasoning.
Phase 2 — Surface reasoning: Now show your step-by-step reasoning.
Phase 3 — Deep scrutiny: Critique your own Phase 2 reasoning. Find any leaps, assumptions, or errors.
Phase 4 — Final answer: Synthesize Phases 1-3 into a refined answer, noting where your quick answer differed from your reasoned answer.

After all phases:
- What did the quick answer miss that deep reasoning caught?
- What did deep reasoning overcomplicate that the quick answer got right?
```
> 📅 2026-07-12
> 来源：Reasoning / Prompt Engineering (2026)
> 用法：链式思维"退火"——从快速直觉到深层推理再到自我批评，适合需要多层次思考的复杂问题。

## 6. 提示词精简压缩术

**Prompt：**
```
Compress this prompt without losing key information.

Original prompt:
[粘贴长提示词]

Target: reduce token count by at least 40% while preserving:
1. Core instruction
2. Output format constraints
3. Key contextual details
4. Quality guardrails

For each compression technique used:
- Flag what was changed
- Explain why it won't affect quality
- Show the before/after token count

Compression techniques to consider:
- Remove redundant adjectives and filler words
- Replace long descriptions with concise directives
- Use delimiter shorthand (### instead of "The following is:")
- Merge related constraints
- Remove polite phrasing (no need for "please" with AI)
- Consolidate examples (use one strong example instead of three weak ones)
```
> 📅 2026-07-12
> 来源：Prompt Optimization / AI Engineering (2026)
> 用法：提示词减肥——在不牺牲质量的前提下减少40%+ token用量，节省成本和延迟。

## 7. 输出 Schema 强制验证器

**Prompt：**
```
Validate and enforce the output schema for my AI response.

Required output format:
[粘贴 Schema — JSON/YAML/CSV/结构化文本]

Input context:
[粘贴上下文或任务描述]

Task:
1. Parse my required output schema
2. Generate the response strictly according to this schema
3. After generation, validate the output against the schema:
   - Are all required fields present?
   - Are data types correct?
   - Are value ranges within allowed limits?
   - Are there any extra fields that shouldn't be there?
4. If validation fails, auto-correct and re-validate
5. Report: what was validated, what was corrected (if anything)

Strict mode: Do NOT add any fields not in the schema. Do NOT omit required fields.
If the schema cannot be satisfied with the given context, list the missing data needed.
```
> 📅 2026-07-12
> 来源：Output Control / Prompt Engineering (2026)
> 用法：输出Schema强制验证器——先按要求生成再自我校验，不满足Schema则自动修正或报告缺失。

## 10. 结构化 JSON 输出强制模式

**Prompt：**
```
Output ONLY valid JSON. Do not include any text outside the JSON object.
No markdown code fences, no explanations, no greetings.

Schema:
{
  "data": [
    {
      "field_name": "description of field",
      "field_type": "string|number|boolean|array|object",
      "required": true|false,
      "validation_rules": "any constraints"
    }
  ],
  "metadata": {
    "record_count": "number",
    "generated_at": "ISO timestamp",
    "version": "string"
  }
}

Input data: [粘贴输入数据]

RULES:
- Output ONLY a valid JSON object
- All required fields MUST be present
- Use null (not "N/A" or empty string) for missing optional fields
- Strings in double quotes, numbers without quotes
- Validate the JSON is parseable before responding
- If input is invalid, return: {"error": "description of the issue"}
```
> 来源：AI Prompt Architect — Structured Output 2026
> 用法：强制AI输出精确符合schema的JSON，用于程序化调用的生产环境。---
