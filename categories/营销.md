## 1. 营销策略框架

**Prompt：**
```
Create a marketing strategy for:

Product/Service: [产品描述]
Target market: [目标市场]
Budget: [预算范围]
Timeline: [时间周期]
Current traction: [现有用户/收入]

Framework:
1. Market analysis (TAM, SAM, SOM)
2. Target personas (3 segments max)
3. Positioning statement
4. Channel strategy (which channels and why)
5. Content pillars (4-5 themes)
6. Launch tactics (first 90 days)
7. Success metrics with targets
```
> 来源：AiledGrowth — ChatGPT Prompts for Marketing 2026
> 用法：完整营销策略框架，从市场分析到执行计划。

## 2. 广告文案生成（Google/FB）

**Prompt：**
```
Write ad copy for:

Platform: [Google Ads / Facebook Ads / LinkedIn Ads]
Product: [产品]
Audience: [定向受众]
Goal: [点击/转化/品牌曝光]

Generate:
- 5 headlines (max 30 chars for Google)
- 3 descriptions (max 90 chars)
- 2 calls-to-action

For each, include:
- Emotional trigger
- Value proposition
- Urgency element (if applicable)
- Predicted CTR ranking

Note which emotional triggers each variant uses.
```
> 来源：AiledGrowth — Marketing Prompts 2026
> 用法：广告文案多版本生成，含情感触发因素分析。

## 3. 内容日历规划

**Prompt：**
```
Plan a content calendar for:

Channel: [博客/Newsletter/社交媒体]
Topic cluster: [主题领域]

Generate:
1. Theme for each week/month

3. Content format (article/video/infographic)

```
> 来源：ClickUp / AiledGrowth 2026
> 用法：内容日历规划，含重新利用策略（一鱼多吃）。

## 4. 销售跟进邮件序列

**Prompt：**
```
Write a sales follow-up sequence for:

Lead type: [潜客类型]
Initial contact: [首次接触方式]
Product: [产品/服务]
Buyer persona: [决策者画像]

Email sequence (5 emails):
1. Day 1 — Value-add follow-up (not just checking in)
2. Day 3 — Case study or social proof
3. Day 7 — Specific use case relevant to them
4. Day 14 — Offer a demo or trial
5. Day 21 — Breakup email (polite, with one last reason)

Each email: subject line (max 50 chars), body (max 100 words), single CTA.
```
> 来源：AiledGrowth — Sales Prompts 2026
> 用法：完整的销售跟进序列，从触达到分手邮件。

## 6. 解锁邮件营销

**Prompt：**
```
Write a re-engagement email for:

Subscriber segment: [不活跃时间段]
Original opt-in reason: [订阅原因]
Value they got before: [之前的价值]

The email should:
1. Acknowledge the silence (not ignore it)
2. Provide a compelling reason to re-engage
3. Make it easy to say "still interested" (single click)
4. Include an unsubscribe option prominently
5. Offer an incentive if appropriate

Subject line options (3 variants).
```
> 来源：Nigape — AI Prompt Examples 2026
> 用法：重新激活沉寂用户的邮件，承认沉默并给予重新参与的理由。

## 7. 用户反馈 → 产品需求文档

**Prompt：**
```
Convert these user feedback items into a product requirements document:

User feedback: [粘贴用户反馈]

Structure:
1. Problem statement (from user perspective)
2. User stories (as a [user], I want [goal], so that [reason])
3. Success metrics (how we measure this solves the problem)
4. Proposed solution (high-level)
5. Scope: in-scope vs out-of-scope
6. Technical considerations
7. UX requirements
8. Priority ranking (P0/P1/P2)
```
> 来源：SurePrompts — Feature Specification 2026
> 用法：把零散的用户反馈转化为可执行的 PRD。

## 9. LinkedIn 广告文案（三种风格）
> 📅 2026-07-05

**Prompt：**
```
Generate 3 very different LinkedIn Ad copies for: [产品/服务描述]. Avoid "look no further".

Ad 1: Question-Based Opening — begin with 2-3 engaging questions.
Ad 2: Targeted and Humorous — start with "Attention [target persona]" and use creative humor.
Ad 3: Long Copy — use problem-solution or AIDA framework.

For each, provide a creative CTA.
```
> 来源：resonatehq.com — LinkedIn Ads Generator (2026)
> 用法：LinkedIn 广告多版本生成，覆盖不同创意角度进行测试。

## 12. 产品发布检查清单
> 📅 2026-07-05

**Prompt：**
```
Create a product launch checklist for: Product: [产品名称]. Launch type: [软发布/公测/正式发布]. Channels: [渠道列表]. Team size: [人数]. Include phases: (1) Pre-launch (T-30 days): final testing, pricing, assets; (2) Launch week: go-live sequence, monitoring; (3) Post-launch (T+30): feedback collection, iteration. For each item: task, owner, deadline, status (pending/in progress/done). Mark critical path items.
```
> 来源：SurePrompts — Product Launch Checklist (2026)
> 用法：产品发布全流程检查清单，含关键路径标识。

## 13. 客户画像构建
> 📅 2026-07-05

**Prompt：**
```
Build a detailed customer persona for: Product: [产品名称]. Market segment: [细分市场]. Available data: [用户调研/访谈/分析数据]. Provide: (1) Demographics (age, role, company size, location), (2) Job responsibilities and daily tasks, (3) Goals and metrics they're measured on, (4) Pain points and frustrations, (5) Decision criteria for purchasing, (6) Information sources they trust, (7) Objections they might raise. Give the persona a name and a quote.
```
> 来源：saascrmreview.com — Buyer Persona Builder (2026)
> 用法：基于已知数据构建详细的客户画像，含姓名和引语增加可记忆性。

## 14. 客户反对意见应对话术
> 📅 2026-07-05

**Prompt：**
```
Prepare sales objection handling scripts for: Product: [产品描述]. Common objections I face: [列举客户常见反对意见]. For each objection, provide: (1) empathetic acknowledgment, (2) clarifying question to understand the real concern, (3) structured response addressing the core issue, (4) proof point or case study reference, (5) trial close question. Tone: consultative, not pushy.
```
> 来源：saascrmreview.com — Sales Objection Handling (2026)
> 用法：销售反对话术准备，每个反对意见提供从共情到锁定的一整套流程。
