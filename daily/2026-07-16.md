## 1. 品牌起源故事轮播

**Prompt：**

## 52. 成功标准提示法（Success Criteria Prompting）

**Prompt：**
```
Before answering, define what a good answer looks like for this task. Specify:
1. Who the output is for
2. What 'good' means in this context (clear, actionable, beginner-friendly, etc.)
3. How the output will be judged
4. What to avoid

Then proceed with the task: [task description]
```
> 来源：Craig Does AI — Success Criteria Prompting 2026
> 用法：先定义"好答案"的标准再执行，大幅提升输出相关性。

---

## 53. 去AI味改写指令

**Prompt：**
```
请将以下文本改写得更口语化，加入个人经历和情绪波动，避免使用"首先、其次、最后"等结构词，让内容看起来像是真人熬夜赶工写出来的。保留所有关键信息，但增加"人类瑕疵"——比如偶尔跑题、用错成语、带点小情绪。
```
> 来源：AtomGit/CSDN — 2026最火爆AI提示词
> 用法：绕过AIGC检测的有效方法，让AI输出更像人类手写。

## 54. 逆向提示工程（Reverse Prompting）

**Prompt：**
```
I need a prompt for the following task. First, ask me clarifying questions about the goal, audience, format, and constraints. Then, write a complete prompt I can copy-paste. Also explain why each part is structured that way so I can learn to do it myself.

Task: [描述你的需求]
```
> 来源：Ali H. Salem — Prompt Engineering Guide 2026
> 用法：让AI先问你问题，再帮你写出完整提示词——同时教你为什么这么写。

## 55. 合同式系统提示词

**Prompt：**
```
CONTRACT

ROLE: [角色定义，如 Senior Data Analyst]
TASK: [具体任务描述]
SUCCESS CRITERIA:
- [标准1: 可衡量]
- [标准2: 可衡量]
- [标准3: 可衡量]

OUTPUT FORMAT: [指定格式]

CONSTRAINTS:
- [约束1]
- [约束2]
- [约束3]

When you understand this contract, say "Contract accepted" and begin.
```
> 来源：Prompt Builder — Claude Prompt Engineering Best Practices 2026
> 用法：合同式系统提示词，Claude 上表现最佳的结构——明确角色、任务、成功标准和约束。
