## 1. 成功标准提示法（Success Criteria Prompting）

**Prompt：**
```
Goal: {{one sentence describing what you want}}

Success criteria:
- {{must-have 1}}
- {{must-have 2}}
- {{must-not-do}}

Constraints:
- {{hard rule 1}}
- {{hard rule 2}}

Output format:
- Section 1: {{name}} ({{length constraint}})
- Section 2: {{name}} ({{length constraint}})
```
> 来源：Craig Does AI / PromptBuilder.cc — Success Criteria Prompting 2026
> 用法：先定义"好答案长什么样"再让 AI 输出，比模糊说"make this better"效果好得多。

## 2. 四块式提示词模板（4-Block Layout）

**Prompt：**
```
## INSTRUCTIONS
{{what to do — the core task}}

## INPUTS
{{the data, document, or context the AI should use}}

## CONSTRAINTS
{{scope, exclusions, uncertainty handling rule}}

## OUTPUT FORMAT
{{the contract — format, length, tone, sections}}
```
> 来源：PromptBuilder.cc — Prompt Engineering Best Practices 2026
> 用法：将指令、输入、约束、输出格式区分成四个独立区块，比混在一起写更清晰、更可调试。

## 3. AI 写作去味指令（去除 AI 腔）

**Prompt：**
```
Rewrite the following text to sound like a human wrote it. Specifically:
- No phrases like "in today's fast-paced world", "it's important to note", "in conclusion", "delve into", "landscape of", "navigate the"
- No bulleted lists unless the original had them
- Vary sentence length. Short sentences are fine.
- Use contractions naturally.
- Cut any sentence that's just scaffolding — if removing it changes nothing, it goes.

Text:
[paste]
```
> 来源：NovaKit.ai — 25 Prompt Engineering Templates That Actually Work in 2026
> 用法：明确列出 AI 最常用的陈词滥调和空话，针对性去掉，比说"写得自然点"有效得多。

## 5. 编辑模式（只改不写）

**Prompt：**
```

Read this draft and reply in three sections:
1. Three lines/paragraphs to cut and why (be blunt).
2. Three unclear claims that need evidence or concreteness.
3. One thing this draft is missing that a reader would expect.

Do not rewrite anything yourself.

Draft:
[paste]
```

## 23. 苏格拉底式导师

**Prompt：**
```

- Ask me one question at a time.
- Wait for my answer before continuing.

- When I get something right, explain why it's right — don't just say "correct."

## 40. A/B 测试计划生成器

**Prompt：**
```
Design an A/B test for:

Element to test: [headline / CTA button / pricing display / landing page layout]
Current version: [describe current]
Proposed variation: [describe proposed variation]
Primary metric: [conversion rate / click-through rate / revenue per visitor]
Traffic: [estimated daily visitors to the test page]

Provide:
1. Hypothesis statement (structured: If we change X, then Y will happen because Z)
2. Minimum sample size calculation (use baseline conversion rate of [X%] and minimum detectable effect of [Y%])
3. Estimated test duration
4. Success and secondary metrics
5. Guardrail metrics (what shouldn't change)
6. Segmentation plan (desktop vs mobile, new vs returning)
7. Decision rule (when and how to declare a winner)
